B Stott

8 months ago  -  Shared publicly
 
I read a set of AT commands for the HC-05 and they stated that unless you change the AT+CMODE=<param> from default '1' to '0' then the specified address will not work. The default CMODE '1' allows the device to bind to any Bluetooth. CMODE '0' sets the Bluetooth module to bind to a specific address - only. [http://elecfreaks.com/store/download/datasheet/Bluetooth/HC-0305%20serail%20module%20AT%20commamd%20set%20201104%20revised.pdf ]

I am in the midst of setting up two ZS-040 modules (similar to the older HC-05) and may not finish before I forget to post here the CMODE command for others to use. The ZS-040 has an additional feature on the pcb from the HC-05. It is a command button which looks like a reset. This command button needs to be held down while you connect the board power pin [EN] to the Arduino pin 9 to get the Buetooth module into AT command mode. As Phillip, I am using techbitnar's code for programming the Bluetooth modules.

Interesting side note for my module: I can not retrieve the NAME from the Bluetooth module using AT+NAME?. If I set the name using AT+NAME=<param> I receive the expected 'OK' but, can not retrieve the name. Other commands such as: UART?, PSWD?, CMODE?, and BIND? work as expected﻿
